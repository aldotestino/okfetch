---
title: Choose the Right Package
description: Pick the package that matches the amount of structure you need.
---

`@okfetch/fetch` is the base package. `@okfetch/api` builds on it when you want endpoint definitions. `@okfetch/logger` is only useful as a plugin inside `@okfetch/fetch`.

| If you need to... | Use |
| --- | --- |
| Make a few requests directly | `@okfetch/fetch` |
| Centralize many endpoints behind one client | `@okfetch/api` |
| Add logging through `plugins` on fetch requests | `@okfetch/logger` |

- Start with `@okfetch/fetch`.
- Move to `@okfetch/api` when request shapes start repeating.
- Add `@okfetch/logger` only when you want ready-made request logs.
